# portbroker API Documentation — Design Spec

**Date:** 2026-03-29
**Extends:** `docs/superpowers/specs/2026-03-29-portbroker-design.md`

---

## Overview

Add interactive API documentation to portbroker via two new daemon endpoints and a `portctl gendocs` subcommand for regenerating the spec on demand.

- `GET /docs` — Redoc-powered interactive API reference, served by the daemon
- `GET /openapi.yaml` — static OpenAPI 3.0 spec served from disk
- `portctl gendocs` — regenerates `openapi.yaml` from `gendocs.py`

---

## Architecture

Three moving parts with single responsibilities:

1. **`portbroker/openapi.yaml`** — source of truth for the API spec. Static file committed to the repo, generated and regenerated by `portctl gendocs`. Never edited by hand.

2. **`portbroker/gendocs.py`** — standalone Python script that writes `openapi.yaml`. Kept separate from `server.py` so the daemon has no dependency on doc generation logic.

3. **Daemon additions to `server.py`** — two new GET routes:
   - `GET /openapi.yaml` — reads and serves the static file from disk
   - `GET /docs` — serves a self-contained HTML page loading Redoc from CDN, pointed at `/openapi.yaml`

## OpenAPI Spec Structure

**File:** `portbroker/openapi.yaml`

```
openapi: 3.0.3
info:
  title: portbroker
  version: 1.0.0
  description: Local port assignment daemon for Docker Compose projects and worktrees.

servers:
  - url: http://localhost:9876

paths:
  /health
  /assignments
  /assignments/{project}
  /assignments/{project}/{worktree}
  /register
  /deregister
  /cleanup

components/schemas:
  Assignment
  AssignmentMap
  RegisterRequest
  DeregisterRequest
  Error
```

### Schemas

**Assignment**
```yaml
type: object
properties:
  project:       { type: string }
  worktree:      { type: string }
  ephemeral:     { type: boolean }
  registered_at: { type: string, format: date-time }
  ports:         { type: object, additionalProperties: { type: integer } }
required: [project, worktree, ephemeral, registered_at, ports]
```

**AssignmentMap**
```yaml
type: object
properties:
  assignments:
    type: object
    additionalProperties: { $ref: '#/components/schemas/Assignment' }
```

**RegisterRequest**
```yaml
type: object
properties:
  project:   { type: string }
  worktree:  { type: string, default: main }
  ephemeral: { type: boolean, default: false }
  services:  { type: array, items: { type: string }, minItems: 1 }
required: [project, services]
```

**DeregisterRequest**
```yaml
type: object
properties:
  project:  { type: string }
  worktree: { type: string, default: main }
required: [project]
```

**Error**
```yaml
type: object
properties:
  error: { type: string }
required: [error]
```

### Routes

| Method | Path | Summary | Response codes |
|--------|------|---------|----------------|
| GET | `/health` | Daemon liveness check | 200 |
| GET | `/assignments` | List all assignments | 200 |
| GET | `/assignments/{project}` | List all worktrees for a project | 200 |
| GET | `/assignments/{project}/{worktree}` | Get one assignment | 200, 404 |
| POST | `/register` | Register a project/worktree and assign ports | 200, 400, 409 |
| DELETE | `/deregister` | Release a project/worktree | 200, 400, 403, 404 |
| POST | `/cleanup` | Remove all ephemeral entries | 200 |

Each path entry includes:
- A one-line `summary`
- A multi-line `description` with a `curl` example
- Full request body schema (where applicable)
- All documented response codes with `$ref` to schemas

The `/docs` and `/openapi.yaml` routes are excluded from the spec — they are infrastructure, not API.

## Daemon Changes (`server.py`)

Two new branches added to `do_GET` in `PortBrokerHandler`:

```python
elif self.path == '/openapi.yaml':
    self._serve_openapi()

elif self.path == '/docs':
    self._serve_docs()
```

**`_serve_openapi()`** — reads `openapi.yaml` from the same directory as `server.py` and serves it with `Content-Type: application/yaml`. Returns 404 with a plain-text error if the file is missing (e.g., `gendocs.py` has never been run).

**`_serve_docs()`** — returns the Redoc HTML string (generated by `_build_docs_html()`, analogous to the existing `_build_ui_html()`).

### `_build_docs_html()`

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>portbroker API</title>
  <style>
    body { margin: 0; padding: 0; background: #1e1e1e; }
  </style>
</head>
<body>
  <div id="redoc-container"></div>
  <script src="https://cdn.jsdelivr.net/npm/redoc/bundles/redoc.standalone.js"></script>
  <script>
    Redoc.init('/openapi.yaml', {
      theme: {
        colors: { primary: { main: '#569cd6' } },
        typography: { fontFamily: 'monospace, monospace' },
        sidebar: { backgroundColor: '#1e1e1e', textColor: '#d4d4d4' },
        rightPanel: { backgroundColor: '#252525' }
      }
    }, document.getElementById('redoc-container'))
  </script>
</body>
</html>
```

Redoc does not expose every internal color token, so the match to the existing UI will be approximate — primary accent, sidebar, and right panel background are controllable; the main content area will use Redoc's defaults with a dark sidebar.

## `portctl gendocs`

New subcommand added to `portctl`:

```bash
portctl gendocs
```

**Behavior:**
- Resolves the path to `gendocs.py` relative to `portctl`'s own directory (`$(dirname "$0")/gendocs.py`)
- Runs `python3 <path>/gendocs.py`
- `gendocs.py` writes `openapi.yaml` to the same directory as itself and prints the absolute path
- Works from any working directory

**`gendocs.py`** is a standalone Python 3 script (no imports beyond stdlib) that constructs the full OpenAPI document as a Python dict and writes it as YAML using only `json` (converted via a minimal YAML serializer) — or uses `import yaml` if PyYAML is available, falling back to a simple dict-to-YAML formatter if not. Since the spec structure is predictable and shallow, a stdlib-only fallback is straightforward.

## Tests

Two new test methods added to `TestAPIHealth` (or a new `TestAPIDocs` class) in `tests/test_api.py`:

- `test_docs_returns_200` — `GET /docs` returns 200 with `Content-Type: text/html`
- `test_openapi_yaml_returns_200` — `GET /openapi.yaml` returns 200 with `Content-Type: application/yaml`
- `test_openapi_yaml_missing_returns_404` — if `openapi.yaml` doesn't exist, `/openapi.yaml` returns 404

The third test requires starting a server pointed at a temp directory that has no `openapi.yaml`.

---

## Constraints

- `gendocs.py` uses stdlib only (no PyYAML required). YAML serialization is handled inline since the spec structure is predictable.
- The daemon does not auto-regenerate `openapi.yaml` on startup — it serves whatever is on disk.
- If `/openapi.yaml` is missing, `GET /docs` will still return 200 (the HTML loads fine) but Redoc will show a load error in the browser. This is acceptable — the fix is `portctl gendocs`.
- CDN dependency: Redoc is loaded from `cdn.jsdelivr.net`. No internet = no docs page rendering. This is acceptable for a local dev tool.
